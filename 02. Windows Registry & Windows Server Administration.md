# 02. Windows Registry & Windows Server Administration

## 📘 Overview

This document provides guidance for safely navigating, editing, and hardening the Windows Registry, along with performing essential Windows Server administration tasks.

The procedures outlined in this guide are intended for IT administrators, systems engineers, and infrastructure support personnel managing Windows Server environments.

---

# 2.1 Windows Registry Essentials

## 🎯 Objective

Safely manage and modify Windows Registry configurations while maintaining system stability and security.

---

## 🛠️ Step 1: Open Registry Editor

### Launch Registry Editor

Run the following:

```text id="4m64aq"
regedit
```

When prompted by User Account Control (UAC), select:

```text id="4mv7wl"
Yes
```

> ⚠️ Administrative privileges are required to modify protected registry keys.

---

## 🛠️ Step 2: Key Hives Overview

### Important Registry Hives

| Registry Hive                            | Description                                        |
| ---------------------------------------- | -------------------------------------------------- |
| `HKLM\SOFTWARE`                          | Machine-wide application settings                  |
| `HKCU\SOFTWARE`                          | Per-user application settings                      |
| `HKLM\SYSTEM\CurrentControlSet\Services` | Service configuration settings                     |
| `HKLM\SECURITY`                          | Local security policy settings (restricted access) |

---

## 🛠️ Step 3: Create a Registry Backup

### Export the Registry Before Changes

Navigate to:

```text id="az4n4m"
File → Export → All
```

Save the backup file using the following naming format:

```text id="5j6r1v"
backup_YYYYMMDD.reg
```

> ✅ Always create a registry backup before making any modifications.

---

## 🛠️ Step 4: Common Admin Edits

### Disable USB Storage

Registry Path:

```text id="3l5hxt"
HKLM\SYSTEM\CurrentControlSet\Services\USBSTOR
```

Modify the following value:

```text id="uxy6fb"
Start = 4
```

> 🔒 This disables USB storage device access on the system.

---

### Configure Legal Logon Banner

Registry Path:

```text id="5wv4rg"
HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System
```

Create or modify the following registry values:

#### LegalNoticeCaption

```text id="cq2z85"
(REG_SZ) = 'Authorised Users Only'
```

#### LegalNoticeText

```text id="6z0miy"
(REG_SZ) = 'Unauthorised access is prohibited.'
```

> 🛡️ Legal banners help reinforce acceptable use and security policies.

---

## 🛠️ Step 5: Remote Registry Edit

### Execute Remote Registry Command

```powershell id="n2uq7i"
reg add \\RemotePC\HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate /v DisableWindowsUpdateAccess /t REG_DWORD /d 0 /f
```

### Parameter Overview

| Parameter | Description                          |
| --------- | ------------------------------------ |
| `/v`      | Registry value name                  |
| `/t`      | Registry data type                   |
| `/d`      | Registry value data                  |
| `/f`      | Force overwrite without confirmation |

> ⚠️ Ensure Remote Registry services and firewall rules are properly configured before executing remote modifications.

# 2.2 Windows Server Core Administration Tasks

## 🎯 Objective

Perform essential administrative tasks required for maintaining and monitoring Windows Server environments.

---

## 🛠️ Step 1: Disk Management

### Open Disk Management

Navigate to:

```text id="71ah8c"
Server Manager → Tools → Disk Management
```

### Administrative Tasks

Use Disk Management to:

* Initialize disks
* Create partitions
* Format new volumes
* Assign drive letters
* Extend or shrink partitions

---

## 🛠️ Step 2: Event Viewer

### Open Event Viewer

Navigate to:

```text id="kbxg58"
Server Manager → Tools → Event Viewer
```

### Review Windows Logs

Inspect the following logs regularly:

| Log Type      | Purpose                              |
| ------------- | ------------------------------------ |
| `System`      | Operating system and hardware events |
| `Application` | Application-generated events         |
| `Security`    | Authentication and audit events      |

---

## 🛠️ Step 3: Performance Monitor

### Launch Performance Monitor

Run:

```text id="e0uc92"
PerfMon
```

### Start System Performance Monitoring

Navigate to:

```text id="qft6eb"
Data Collector Sets → System Performance → Start
```

> 📊 Performance Monitor assists with resource utilization analysis and troubleshooting.

---

## 🛠️ Step 4: PowerShell — Common Snippets

### Retrieve Recent System Errors

```powershell id="d5i0f2"
Get-EventLog -LogName System -Newest 50 -EntryType Error
```

### List Stopped Services

```powershell id="1zfdq2"
Get-Service | Where-Object {$_.Status -eq 'Stopped'}
```

### Restart Print Spooler Service

```powershell id="2ey2sr"
Restart-Service -Name 'Spooler'
```

### Display Top CPU-Consuming Processes

```powershell id="30n46p"
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10
```

# ✅ Best Practice Tips

## 🔒 Security Best Practices

* Always export the registry before making changes.
* Restrict `regedit` access via Group Policy Objects (GPO) for standard users.
* Use Group Policy Preferences instead of direct registry modifications whenever possible.
* Limit administrative access to critical registry hives.

---

## 💾 Backup & Recovery Recommendations

* Enable the Windows Server Backup role for daily system-state backups.
* Store backups on separate storage media or network locations.
* Regularly validate backup restoration procedures.

---

## 📊 Monitoring & Logging Recommendations

* Keep Event Logs sufficiently large.

Recommended minimum size:

```text id="sj8bxn"
256 MB
```

* Forward critical logs to a Security Information and Event Management (SIEM) platform.
* Monitor Performance Monitor counters regularly for capacity planning and troubleshooting.

---

## ⚙️ Operational Recommendations

* Test registry modifications in a non-production environment before deployment.
* Document all administrative registry changes.
* Use PowerShell scripting for repeatable administrative tasks.
* Apply least-privilege access principles across server environments.
