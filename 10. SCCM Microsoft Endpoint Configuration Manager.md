# 10. SCCM / Microsoft Endpoint Configuration Manager

## 📘 Overview

This document provides enterprise-level guidance for deploying and managing Microsoft Endpoint Configuration Manager.

The guide covers:

* Primary site installation
* Client deployment
* Software distribution
* Software Update Point (SUP) configuration
* Patch management workflows
* Best practices for hybrid management

---

# 10.1 SCCM Primary Site Installation (Overview)

## 🎯 Objective

Deploy the SCCM primary site server and prepare the environment for enterprise device management.

---

## 🛠️ Step 1: Prerequisites

### Required Infrastructure Components

| Component          | Requirement                  |
| ------------------ | ---------------------------- |
| Operating System   | Windows Server 2019 or later |
| Database           | SQL Server 2019 or later     |
| Deployment Toolkit | Windows ADK + WinPE Add-on   |

---

## Active Directory Preparation

### Extend AD Schema

Run:

```text id="sccm01"
extadsch.exe
```

> 📘 This prepares Active Directory for SCCM integration.

---

### Create System Management Container

* Create **System Management** container in Active Directory
* Assign **Full Control** permissions to SCCM site server

> 🔐 This enables SCCM to publish and manage site information in AD.

---

## 🛠️ Step 2: Run SCCM Setup

### Launch Installer

Mount SCCM installation media and run:

```text id="sccm02"
splash.hta
```

---

### Installation Configuration

| Setting      | Description                 |
| ------------ | --------------------------- |
| Site Type    | Primary Site                |
| Site Code    | Example: `PS1`              |
| Site Name    | Custom organization name    |
| Install Path | SCCM installation directory |

---

### Installation Process

* Complete prerequisite checks
* Proceed with installation
* Monitor setup completion

> ⚙️ A successful installation establishes the SCCM site hierarchy.

---

# 10.2 Client Deployment

## 🎯 Objective

Deploy SCCM clients to managed endpoints for centralized control.

---

# 🛠️ Step 1: Enable Client Push Installation

## Configure Client Installation Settings

Navigate to:

```text id="sccm03"
SCCM Console → Administration → Site Configuration → Sites → Client Installation Settings
```

---

## Enable Client Push

* Enable **Client Push Installation**
* Provide required installation credentials

> 🔄 Enables automatic deployment of SCCM agents to target devices.

---

## Manual Client Installation

Navigate to:

```text id="sccm04"
Devices → Right-click Device → Install Client
```

---

# 10.3 Software Distribution

## 🎯 Objective

Deploy applications and software packages across enterprise devices.

---

# 🛠️ Step 1: Create Application

Navigate to:

```text id="sccm05"
Software Library → Application Management → Applications → Create Application
```

---

## Application Deployment Methods

| Method            | Description                     |
| ----------------- | ------------------------------- |
| MSI Detection     | Automatic application packaging |
| Manual Definition | Custom deployment configuration |

---

> 📦 Applications in SCCM support versioning, detection rules, and dependencies.

---

# 🛠️ Step 2: Deploy Application

## Deploy to Device Collections

Right-click application and select:

```text id="sccm06"
Deploy
```

---

## Deployment Options

| Option    | Description                      |
| --------- | -------------------------------- |
| Required  | Forced installation              |
| Available | Self-service via Software Center |

> ⚠️ Required deployments enforce installation without user interaction.

---

# 10.4 Software Update Management (SUP)

## 🎯 Objective

Manage Windows and Microsoft product patching using SCCM Software Update Point.

---

# 🛠️ Step 1: Configure Software Update Point

Navigate to:

```text id="sccm07"
Administration → Site Configuration → Servers → Site System Roles
```

---

## Add SUP Role

* Add **Software Update Point**
* Configure synchronization source:

```text id="sccm08"
Microsoft Update
```

---

## Select Products

Include:

* Windows 10 / Windows 11
* Microsoft Office 365

> 🔄 Ensures update catalog is synchronized with Microsoft Update services.

---

# 🛠️ Step 2: Deploy Patch Tuesday Updates

## Filter Updates

Navigate to:

```text id="sccm09"
Software Library → Software Updates → All Software Updates
```

Filter:

* Current month updates

---

## Deployment Process

1. Download updates
2. Create deployment package
3. Deploy to target collection

---

## Maintenance Window Configuration

| Setting | Example          |
| ------- | ---------------- |
| Day     | Saturday         |
| Time    | 02:00 – 05:00 AM |

> 🕒 Maintenance windows prevent disruption during production hours.

# ✅ Best Practice Tips

## 🧩 Device Management Best Practices

* Use Active Directory-based collections for dynamic targeting
* Organize devices by role, department, or location

---

## 🧪 Patch Management Best Practices

* Always test updates in a pilot collection before production rollout
* Schedule deployments using maintenance windows

---

## 📊 Operational Governance

* Archive deployment reports for audit and compliance tracking
* Monitor failed deployments and remediation status regularly

---

## ☁️ Modern Management Integration

* Integrate Microsoft Intune with SCCM using co-management
* Gradually transition workloads to cloud-based management

---

## 🔐 Security Best Practices

* Restrict SCCM administrative access using RBAC
* Secure site servers and database access
* Regularly update SCCM infrastructure components
