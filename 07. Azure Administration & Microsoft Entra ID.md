# 07. Azure Administration & Microsoft Entra ID

## 📘 Overview

This document provides guidance for configuring and managing Microsoft Azure resources, identity services, role-based access control (RBAC), and security policies using:

* Microsoft Azure
* Microsoft Entra ID

The guide covers:

* Azure subscription and resource group management
* RBAC role assignments
* Entra ID user and group configuration
* Conditional Access policies
* Azure virtual machine deployment and connectivity

---

# 7.1 Azure Subscription & Resource Groups

## 🎯 Objective

Create and manage Azure resource groups and assign access permissions using RBAC.

---

## 🛠️ Step 1: Create Resource Group

### Azure CLI Command

```bash id="azrg01"
az group create --name RG-Production --location australiaeast
```

---

## Description

| Parameter    | Description                 |
| ------------ | --------------------------- |
| `--name`     | Resource group name         |
| `--location` | Azure region for deployment |

> 📦 Resource groups are logical containers for Azure resources.

---

## 🛠️ Step 2: Assign RBAC Role

### Azure CLI Command

```bash id="azrbac01"
az role assignment create --assignee user@corp.com --role 'Contributor' --scope /subscriptions/<subscription-id>/resourceGroups/RG-Production
```

---

## RBAC Components

| Component | Description                          |
| --------- | ------------------------------------ |
| Assignee  | User or service principal            |
| Role      | Permission level (e.g., Contributor) |
| Scope     | Target resource boundary             |

> 🔐 RBAC ensures controlled and auditable access to Azure resources.

---

# 7.2 Microsoft Entra ID (Azure AD)

## 🎯 Objective

Manage identity lifecycle, group policies, and conditional access using Microsoft Entra ID.

---

# 🛠️ Step 1: Create Entra ID User

## Via Azure Portal

Navigate to:

```text id="entuser01"
Entra ID → Users → New user
```

Fill in required user details.

---

## Via PowerShell

```powershell id="psentra01"
New-AzureADUser -DisplayName 'Jane Smith' -UserPrincipalName jane@corp.com -PasswordProfile $pass -AccountEnabled $true
```

---

## Parameters

| Parameter           | Description                    |
| ------------------- | ------------------------------ |
| `DisplayName`       | User full name                 |
| `UserPrincipalName` | Login identity                 |
| `PasswordProfile`   | Initial password configuration |
| `AccountEnabled`    | Enables user account           |

---

# 🛠️ Step 2: Create Dynamic Group

## Configure Dynamic Membership

Navigate to:

```text id="grp01"
Entra ID → Groups → New Group
```

### Configuration

| Setting         | Value                        |
| --------------- | ---------------------------- |
| Membership Type | Dynamic                      |
| Rule            | `(user.department -eq 'IT')` |

> 🔄 Dynamic groups automatically manage membership based on user attributes.

---

# 🛠️ Step 3: Conditional Access Policy

## Create Policy

Navigate to:

```text id="ca01"
Entra ID → Security → Conditional Access → New Policy
```

---

## Policy Configuration

| Setting       | Value          |
| ------------- | -------------- |
| Users         | All users      |
| Cloud Apps    | All cloud apps |
| Grant Control | Require MFA    |

---

## Deployment Strategy

| Phase   | Action                  |
| ------- | ----------------------- |
| Phase 1 | Enable Report-only mode |
| Phase 2 | Monitor impact          |
| Phase 3 | Switch to Enabled (On)  |

> 🛡️ Conditional Access enhances identity security by enforcing context-aware authentication.

---

# 7.3 Azure Virtual Machines

## 🎯 Objective

Deploy and manage virtual machines within Microsoft Azure Virtual Machines.

---

# 🛠️ Step 1: Create VM via Azure Portal

## Launch VM Creation

Navigate to:

```text id="vm01"
Azure Portal → Create Resource → Virtual Machine
```

---

## VM Configuration

| Setting        | Recommendation        |
| -------------- | --------------------- |
| Region         | Based on proximity    |
| Size           | `B2s` for development |
| Authentication | SSH key or password   |

---

## Networking Configuration

| Setting            | Description          |
| ------------------ | -------------------- |
| Virtual Network    | New VNet + Subnet    |
| Security Rule      | Allow SSH (port 22)  |
| Access Restriction | Limit to own IP only |

> 🌐 Proper network restriction reduces attack surface exposure.

---

# 🛠️ Step 2: Connect to VM

## Linux Connection (SSH)

```bash id="ssh01"
ssh adminuser@<public-ip>
```

---

## Windows Connection (RDP)

```text id="rdp01"
mstsc /v:<public-ip>
```

# ✅ Best Practice Tips

## 🔐 Identity & Access Management

* Use Azure Policy to enforce:

  * Tagging standards
  * Allowed regions
* Apply least privilege principle for all RBAC roles
* Enable Azure Policy across subscriptions

---

## 🛡️ Security Recommendations

* Enable Microsoft Defender for Cloud on all subscriptions
* Use Managed Identities instead of storing credentials
* Enable Privileged Identity Management (PIM) for just-in-time admin access

---

## ⚙️ Governance Best Practices

* Lock critical resource groups using:

```text id="lock01"
CanNotDelete
```

* Monitor identity and access logs regularly
* Review Conditional Access policies periodically

---

## ☁️ Operational Best Practices

* Enforce consistent tagging strategy
* Automate provisioning using infrastructure-as-code tools
* Regularly audit unused resources and orphaned identities
