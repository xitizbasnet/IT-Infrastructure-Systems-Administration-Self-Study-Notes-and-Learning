# 06. VMware vSphere / ESXi Administration

## 📘 Overview

This document provides procedures for deploying and administering a VMware virtualization environment using:

* VMware ESXi
* VMware vCenter Server
* VMware vSphere

The guide includes:

* ESXi host configuration
* vCenter Server Appliance (VCSA) deployment
* VM templates and cloning
* vSphere High Availability (HA)
* Distributed Resource Scheduler (DRS)
* Virtual infrastructure best practices

---

# 6.1 ESXi Host Initial Setup

## 🎯 Objective

Perform the initial configuration of an ESXi hypervisor host.

---

# 🛠️ Step 1: Access ESXi Host Client

## Connect to ESXi Web Interface

Browse to:

```text id="n2j1k9"
https://<ESXi-Host-IP>/ui
```

Log in using:

| Username | Credential         |
| -------- | ------------------ |
| `root`   | ESXi root password |

---

## Configure Management Networking

Navigate to:

```text id="2iyh4f"
Networking → VMkernel NIC
```

Configure the following:

| Setting         | Purpose                    |
| --------------- | -------------------------- |
| Management IP   | Host administration access |
| DNS Server      | Name resolution            |
| Default Gateway | Network routing            |

> 🌐 Proper network configuration is required for vCenter integration and cluster operations.

---

# 🛠️ Step 2: Configure NTP

## Configure Time Synchronization

Navigate to:

```text id="8k4s0g"
Manage → System → Time & Date → NTP Settings
```

Add an NTP server:

```text id="g0dc72"
pool.ntp.org
```

> ⏱️ Accurate time synchronization is critical for authentication, logging, HA, and vMotion operations.

---

# 6.2 vCenter Server Deployment

## 🎯 Objective

Deploy and configure the VMware vCenter Server Appliance (VCSA).

---

# 🛠️ Step 1: Deploy VCSA

## Launch Installer

Mount the VCSA ISO and run the installer from:

* Windows
* Linux
* macOS

---

## Stage 1 — Deploy Appliance

Deploy the appliance OVF to an ESXi host.

### Required Configuration

| Setting          | Description                 |
| ---------------- | --------------------------- |
| Target ESXi Host | Destination hypervisor      |
| Appliance Size   | Based on environment scale  |
| Storage          | Datastore selection         |
| Networking       | Management network settings |

---

## Stage 2 — Configure Appliance

Configure the following:

| Setting                | Example         |
| ---------------------- | --------------- |
| SSO Domain             | `vsphere.local` |
| Administrator Password | Secure password |
| Time Synchronization   | NTP recommended |

> 🔐 Store SSO credentials securely and restrict administrative access.

---

# 🛠️ Step 2: Add ESXi Hosts to vCenter

## Add Host to Inventory

Navigate to:

```text id="ncv0ku"
vSphere Client → Hosts & Clusters → Add Host
```

---

## Provide Host Information

Enter:

* ESXi hostname or IP address
* Root credentials

Accept the host SSL thumbprint when prompted.

> 🔒 Verify SSL thumbprints before acceptance to prevent man-in-the-middle risks.

# 6.3 VM Templates & Cloning

## 🎯 Objective

Standardize VM deployments using templates and automated customization.

---

# 🛠️ Step 1: Convert VM to Template

## Convert Existing VM

Before conversion:

* Power off the virtual machine

Then navigate to:

```text id="0v8w1f"
Right-click VM → Template → Convert to Template
```

> 📦 Templates accelerate consistent VM provisioning.

---

# 🛠️ Step 2: Deploy VM from Template

## Create New VM

Navigate to:

```text id="e0i7r8"
Right-click Template → New VM from this Template
```

---

## Use Customization Specifications

Apply a Customization Specification to automatically configure:

| Setting     | Purpose                      |
| ----------- | ---------------------------- |
| Hostname    | Unique system naming         |
| IP Address  | Network assignment           |
| Domain Join | Active Directory integration |

> ⚙️ Customization Specs reduce manual deployment effort and configuration errors.

# 6.4 vSphere HA & DRS

## 🎯 Objective

Configure high availability and automated workload balancing within a vSphere cluster.

---

# 🛠️ Step 1: Enable HA on Cluster

## Configure vSphere Availability

Navigate to:

```text id="g6d82g"
Cluster → Configure → vSphere Availability → Edit
```

Enable:

```text id="4fc3zu"
vSphere HA
```

---

## Configure Admission Control

Reserve cluster failover capacity:

```text id="lj74yk"
25%
```

### Purpose

Admission Control ensures sufficient resources remain available during host failures.

> 🛡️ HA automatically restarts VMs on surviving hosts after hardware or host outages.

---

# 🛠️ Step 2: Enable DRS

## Configure Distributed Resource Scheduler

Navigate to:

```text id="t9n2n9"
Cluster → Configure → vSphere DRS → Edit
```

Set automation level to:

```text id="zq76r2"
Automatic (3 = Aggressive)
```

---

## DRS Benefits

| Feature                       | Purpose                           |
| ----------------------------- | --------------------------------- |
| Automated VM Placement        | Optimizes resource allocation     |
| Load Balancing                | Prevents host resource exhaustion |
| Dynamic Resource Optimization | Improves cluster performance      |

> ⚡ DRS works alongside vMotion to redistribute workloads dynamically.

# ✅ Best Practice Tips

## 🖥️ Virtual Machine Management Best Practices

* Install VMware Tools on all virtual machines.
* Standardize VM deployments using templates.
* Maintain accurate VM naming and tagging conventions.

---

## ⚙️ Host Configuration Recommendations

* Maintain ESXi Host Profiles for consistent configuration management.
* Apply baseline configurations across all hosts.
* Monitor hardware compatibility and firmware versions regularly.

---

## 🛡️ Disaster Recovery Recommendations

Recommended DR technologies:

* vSphere Replication
* Zerto

### Additional Recommendations

* Test failover procedures regularly.
* Validate replication consistency.
* Maintain offsite backup copies.

---

## 📚 Content Management Recommendations

Use:

vSphere Content Library

for:

* ISO management
* VM template distribution
* Standardized deployment resources

---

## 🔐 Security & Certificate Management Recommendations

* Regularly review vCenter certificates.
* Renew certificates before expiration.
* Restrict root account usage.
* Enable centralized logging and monitoring for ESXi hosts.
