# 01. Active Directory, ADDC, DC, ADUC & GPO

## 📘 Overview

This document provides a step-by-step guide for deploying and configuring a Windows Server Domain Environment using:

* Active Directory Domain Services (AD DS)
* Additional Domain Controllers (ADDC)
* Active Directory Users & Computers (ADUC)
* Group Policy Objects (GPO)

The guide is intended for IT infrastructure and systems administration training and operational environments.

---

# 1. Active Directory Domain Services (AD DS)

## 🎯 Objective

Deploy and configure a Windows Server Domain environment.

---

# 1.1 Install Active Directory Domain Services (AD DS)

## 🛠️ Step 1: Prepare the Server

### Configure Network Settings

* Assign a static IP address
  Example:

```text
192.168.1.10
```

* Set the DNS server address to:

```text
127.0.0.1
```

### Rename the Server

Navigate to:

```text
Server Manager → Local Server → Computer Name
```

Rename the server according to organizational naming standards.

---

## 🛠️ Step 2: Install AD DS Role

### Open Server Manager

Navigate to:

```text
Server Manager → Add Roles & Features
```

### Install Required Role

* Select:

```text
Active Directory Domain Services
```

* Include management tools when prompted.
* Complete the installation wizard.
* Click:

```text
Install
```

---

## 🛠️ Step 3: Promote Server to Domain Controller

### Promote the Server

* Click the notification flag in Server Manager.
* Select:

```text
Promote this server to a domain controller
```

### Configure Deployment

* Choose:

```text
Add a new forest
```

* Enter the Root Domain Name:

```text
corp.local
```

### Configure Domain Controller Options

* Set the DSRM password.
* Keep default settings for:

  * DNS
  * NetBIOS
  * Database paths

### Complete Installation

* Click:

```text
Install
```

* The server will reboot automatically after completion.

---

# 1.2 Add an Additional Domain Controller (ADDC)

## 🛠️ Step 1: Join Secondary Server to Domain

### Configure DNS

Set the DNS server of the secondary server to the IP address of the primary Domain Controller.

### Join the Domain

Navigate to:

```text
System Properties → Change → Domain
```

Enter the domain:

```text
corp.local
```

---

## 🛠️ Step 2: Install AD DS & Promote

### Install AD DS Role

Repeat the AD DS role installation process on the secondary server.

### Promote as Additional Domain Controller

Select:

```text
Add a domain controller to an existing domain
```

### Configure Additional Settings

Enable the following options:

* DNS Server
* Global Catalog

Provide domain administrator credentials and proceed with installation.

---

# 1.3 ADUC — Active Directory Users & Computers

## 🛠️ Step 1: Create Organisational Units (OUs)

### Open ADUC

Run:

```powershell
dsa.msc
```

### Create Organizational Units

Navigate to:

```text
Right-click Domain → New → Organizational Unit
```

Create the following OUs:

* `IT_Dept`
* `HR_Dept`
* `Finance_Dept`

---

## 🛠️ Step 2: Create User Accounts

### Create New Users

Navigate to:

```text
Right-click OU → New → User
```

Fill in the following details:

* First Name
* Last Name
* Logon Name

### Password Configuration

* Set a secure password.
* Uncheck:

```text
User must change password at next logon
```

> ⚠️ Recommended only for service accounts where appropriate.

---

## 🛠️ Step 3: Create Security Groups & Add Members

### Create Security Groups

Navigate to:

```text
Right-click OU → New → Group
```

Select:

* Group Scope: `Global`
* Group Type: `Security`

### Add Group Members

Navigate to:

```text
Group → Members Tab → Add Users
```

---

# 1.4 Group Policy Objects (GPO)

## 🛠️ Step 1: Open Group Policy Management Console

Run the following command from the Domain Controller:

```powershell
gpmc.msc
```

---

## 🛠️ Step 2: Create a New GPO

### Create and Link GPO

Navigate to:

```text
Right-click Target OU → Create a GPO in this domain, and Link it here
```

### Example GPO Name

```text
Password_Policy
```

---

## 🛠️ Step 3: Edit GPO — Password Policy

### Navigate to Policy Settings

```text
Computer Configuration
└── Windows Settings
    └── Security Settings
        └── Account Policies
```

### Configure Password Policy

| Setting                 | Value   |
| ----------------------- | ------- |
| Minimum Password Length | 12      |
| Password Complexity     | Enabled |
| Maximum Password Age    | 90 Days |

---

## 🛠️ Step 4: Edit GPO — Desktop Restrictions (User Config)

### Restrict Control Panel Access

Navigate to:

```text
User Configuration
└── Administrative Templates
    └── Control Panel
```

Set:

```text
Prohibit access = Enabled
```

### Prevent Registry Tools Access

Navigate to:

```text
User Configuration
└── Administrative Templates
    └── System
```

Set:

```text
Prevent Registry Tools = Enabled
```

---

## 🛠️ Step 5: Force Policy Refresh

Run the following command:

```powershell
gpupdate /force
```

# ✅ Best Practice

## 🔒 Security Recommendations

* Use strong administrative passwords.
* Apply the principle of least privilege.
* Regularly back up Active Directory.
* Deploy multiple Domain Controllers for redundancy.
* Monitor GPO changes and domain events.
* Use separate service accounts for applications and services.

## 🧾 Documentation Standards

* Maintain consistent server naming conventions.
* Document all OU structures and GPO configurations.
* Keep inventory records of all Domain Controllers.
* Review Active Directory permissions periodically.

## ⚙️ Operational Recommendations

* Test GPOs in a staging OU before production deployment.
* Use centralized time synchronization.
* Monitor DNS health and replication status.
* Apply Windows updates regularly to all Domain Controllers.
