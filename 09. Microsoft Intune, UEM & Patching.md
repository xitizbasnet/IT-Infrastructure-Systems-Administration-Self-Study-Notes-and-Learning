# 09. Microsoft Intune, UEM & Patching

## 📘 Overview

This document provides guidance for device enrollment, policy deployment, compliance management, and Windows update orchestration using:

* Microsoft Intune
* Microsoft Entra ID

The guide covers:

* Device enrollment (Auto-enrollment & Autopilot)
* Compliance and configuration policies
* Endpoint security baselines
* Windows Update for Business (WUfB)
* Patch compliance monitoring

---

# 9.1 Device Enrolment

## 🎯 Objective

Enrol and register corporate devices into Intune for centralized management and policy enforcement.

---

# 🛠️ Step 1: Auto-Enrolment via Entra ID Join

## Configure MDM Auto-Enrolment

Navigate to:

```text id="intune01"
Entra ID → Mobility (MDM & MAM) → Microsoft Intune
```

Configure:

| Setting        | Value |
| -------------- | ----- |
| MDM User Scope | All   |

> 🔄 This enables automatic device enrollment into Intune when users join devices.

---

## Join Device to Entra ID

On Windows device:

```text id="enroll01"
Settings → Accounts → Access work or school → Connect → Join to Entra ID
```

---

# 🛠️ Step 2: Windows Autopilot (Zero-Touch)

## 🎯 Objective

Deploy devices using zero-touch provisioning via Windows Autopilot.

---

## Retrieve Hardware Hash

### Install Required Script

```powershell id="autopilot01"
Install-Script -Name Get-WindowsAutoPilotInfo
```

---

### Generate Device Hash

```powershell id="autopilot02"
Get-WindowsAutoPilotInfo -OutputFile AutopilotHashes.csv
```

---

## Upload to Intune

Navigate to:

```text id="autopilot03"
Intune → Devices → Enrolment → Windows Autopilot → Import
```

---

## Assign Deployment Profile

Configure OOBE settings:

* User-driven enrollment
* Naming conventions
* Join type (Entra ID Join)

> 🚀 Autopilot enables pre-configured device deployment without manual IT setup.

---

# 9.2 Compliance & Configuration Policies

## 🎯 Objective

Enforce security compliance and standard configuration baselines across managed devices.

---

# 🛠️ Step 1: Create Compliance Policy

Navigate to:

```text id="comp01"
Intune → Devices → Compliance → Create Policy
```

Select:

* Platform: Windows 10/11

---

## Compliance Settings

| Setting            | Requirement |
| ------------------ | ----------- |
| Minimum OS Version | Enforced    |
| BitLocker          | Required    |
| Antivirus          | Required    |

---

## Non-Compliance Actions

| Action                    | Timeline     |
| ------------------------- | ------------ |
| Mark device non-compliant | After 1 day  |
| Send email notification   | After 3 days |

> 🛡️ Compliance policies enforce baseline security posture across all endpoints.

---

# 🛠️ Step 2: Configuration Profile — Endpoint Protection

Navigate to:

```text id="cfg01"
Intune → Devices → Configuration → Create → Templates → Endpoint Protection
```

---

## Enable Security Features

| Feature                      | Status  |
| ---------------------------- | ------- |
| Microsoft Defender Antivirus | Enabled |
| Firewall                     | Enabled |
| SmartScreen                  | Enabled |
| Credential Guard             | Enabled |

> 🔐 These controls strengthen endpoint security and reduce attack surface.

---

# 9.3 Windows Update for Business (WUfB) via Intune

## 🎯 Objective

Manage Windows updates and patch deployment using Intune-based update rings.

---

# 🛠️ Step 1: Create Update Ring

Navigate to:

```text id="wufb01"
Intune → Devices → Update rings for Windows 10 and later → Create
```

---

## Update Ring Configuration

### Pilot Ring

| Setting               | Value   |
| --------------------- | ------- |
| Feature Updates Defer | 30 days |
| Quality Updates Defer | 7 days  |

---

### Production Ring

| Setting               | Value   |
| --------------------- | ------- |
| Feature Updates Defer | 60 days |
| Quality Updates Defer | 14 days |

---

## Assignment Strategy

| Group       | Purpose            |
| ----------- | ------------------ |
| Pilot       | Early testing      |
| All_Devices | Production rollout |

> 🔄 Ring-based deployment reduces patch-related outages.

---

# 🛠️ Step 2: Monitor Patch Compliance

## View Update Reports

Navigate to:

```text id="report01"
Intune → Reports → Windows Updates → Feature Update Report
```

---

## Monitoring Objectives

* Identify non-compliant devices
* Track update adoption rates
* Detect failed patch installations

---

# ✅ Best Practice Tips

## 🔐 Device Management Best Practices

* Always test configuration profiles on a pilot group before full deployment.
* Use Intune Scope Tags to delegate administrative responsibility by region or team.
* Combine Intune compliance with Conditional Access to block non-compliant devices.

---

## 📊 Monitoring & Analytics Recommendations

* Enable Endpoint Analytics
* Track device health and performance baselines
* Monitor compliance drift over time

---

## 🚀 Enrollment & Provisioning Best Practices

* Use Windows Autopilot for all new corporate devices
* Maintain a secure archive of hardware hashes for reprovisioning
* Standardize naming conventions for enrolled devices

---

## 🛡️ Security Best Practices

* Enforce BitLocker encryption across all devices
* Require antivirus and firewall compliance
* Block legacy or unmanaged devices from corporate access
