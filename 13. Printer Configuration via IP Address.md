# 13. Printer Configuration via IP Address

## 📘 Overview

This document provides standardized procedures for configuring and managing network printers using IP-based connections on both Windows and Linux systems.

The guide covers:

* Printer discovery via IP address
* Windows printer installation and sharing
* Linux printing using CUPS
* Windows Print Server configuration
* Printing best practices for enterprise environments

---

# 13.1 Add Printer via IP — Windows

## 🎯 Objective

Install and configure a network printer using a static or DHCP-assigned IP address.

---

## 🛠️ Step 1: Find Printer IP Address

### Methods to Identify Printer IP

* Print a **Configuration Page** from the printer control panel
* Check DHCP server lease table using printer MAC address

> 🖨️ The printer IP is required for network-based installation.

---

## 🛠️ Step 2: Add Printer via Settings

Navigate to:

```text id="winprint01"
Settings → Bluetooth & Devices → Printers & Scanners → Add device
```

---

### Manual Installation (If Not Detected)

Select:

```text id="winprint02"
Add manually → Add a printer using TCP/IP address
```

---

## Printer Configuration

| Setting     | Value                             |
| ----------- | --------------------------------- |
| IP Address  | e.g. 192.168.1.100                |
| Port Name   | Auto-generated (IP_192.168.1.100) |
| Protocol    | Raw or LPR                        |
| Port Number | 9100 (default for most printers)  |

> 📡 Port 9100 (RAW) is the most commonly used printing protocol.

---

## 🛠️ Step 3: Install Printer Driver

* Select from built-in driver list
* Or use **Have Disk** to install vendor-provided driver

> ⚙️ Correct drivers ensure compatibility and print quality.

---

## 🛠️ Step 4: Share Printer via Print Server

Navigate to:

```text id="winprint03"
Printer Properties → Sharing → Share this printer
```

---

### Configuration Options

| Setting     | Description                  |
| ----------- | ---------------------------- |
| Share Name  | Network-visible printer name |
| Permissions | Control user/group access    |

---

## Group Policy Deployment (GPO)

Navigate to:

```text id="winprint04"
Computer Configuration → Preferences → Control Panel Settings → Printers
```

> 🔄 Enables automatic printer deployment to users and computers.

---

## 🛠️ Step 5: Test Print

* Right-click printer
* Select **Print Test Page**

> 🧪 Always validate printer functionality after installation.

---

# 13.2 Add Printer via IP — Linux (CUPS)

## 🎯 Objective

Configure network printers using Common UNIX Printing System (CUPS).

---

## 🛠️ Step 1: Enable CUPS Service

```bash id="cups01"
sudo systemctl enable --now cups
```

---

## Access CUPS Web Interface

```text id="cups02"
http://localhost:631
```

> 🌐 CUPS provides a web-based interface for printer management.

---

## 🛠️ Step 2: Add Printer

Navigate to:

```text id="cups03"
Administration → Add Printer → AppSocket/HP JetDirect
```

---

## Printer URI Configuration

| Setting         | Value         |
| --------------- | ------------- |
| Connection Type | socket        |
| IP Address      | 192.168.1.100 |
| Port            | 9100          |

Example URI:

```text id="cups04"
socket://192.168.1.100:9100
```

---

## Driver Configuration

* Select manufacturer and model
* Or upload **PPD file**

> 🖨️ PPD files define printer capabilities and settings.

---

# 13.3 Print Server — Windows Server

## 🎯 Objective

Deploy centralized print management using Windows Server roles.

---

## 🛠️ Step 1: Install Print Services Role

Navigate to:

```text id="printsrv01"
Server Manager → Add Roles → Print and Document Services → Print Server
```

---

> 🖧 Enables centralized printer hosting and management.

---

## 🛠️ Step 2: Add & Share Printer

Open:

```text id="printsrv02"
Tools → Print Management
```

---

### Configuration Steps

* Add local or remote printer
* Configure drivers
* Right-click printer → **Share**

---

> 📡 Centralized print servers simplify management and policy enforcement.

---

# ✅ Best Practice Tips

## 🌐 Network Printing Standards

* Assign **static IPs** or DHCP reservations to all printers
* Avoid dynamic IP changes for production printers

---

## 📦 Deployment Best Practices

* Deploy printers using Group Policy Preferences
* Map printers based on:

  * User groups
  * Departments
  * Location

---

## 🔐 Security Best Practices

* Restrict printer sharing to authenticated users/groups only
* Disable anonymous printing access
* Monitor print server logs for unauthorized activity

---

## 🖨️ Performance & Reliability

* Use CUPS for mixed Windows/Linux environments
* Enable IPP over HTTPS (Ports 443 / 631) for secure printing
* Regularly update printer firmware and drivers
