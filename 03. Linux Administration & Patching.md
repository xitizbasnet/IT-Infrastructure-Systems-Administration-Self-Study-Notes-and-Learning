# 03. Linux Administration & Patching

## 📘 Overview

This document provides foundational Linux administration procedures for managing Red Hat, Ubuntu, CentOS, and Rocky Linux servers, along with structured operating system patch management practices.

The guide covers:

* User and group administration
* File permissions and ownership
* Service management using `systemd`
* Network troubleshooting commands
* Linux patch management for Ubuntu and RHEL-based distributions
* Security and operational best practices

---

# 3.1 Essential Linux Admin Commands

## 🎯 Objective

Perform essential Linux system administration tasks for server management and maintenance.

---

# 🛠️ Step 1: User & Group Management

## Create a New User

```bash
useradd -m -s /bin/bash john
```

### Description

| Option         | Purpose               |
| -------------- | --------------------- |
| `-m`           | Create home directory |
| `-s /bin/bash` | Assign Bash shell     |

---

## Set User Password

```bash
passwd john
```

---

## Add User to Sudo Group

```bash
usermod -aG sudo john
```

### Description

| Option | Purpose                            |
| ------ | ---------------------------------- |
| `-aG`  | Append user to supplementary group |

---

## Create a New Group

```bash
groupadd developers
```

---

## Verify User Information

```bash
id john
```

### Example Output

```text id="4r4m3m"
uid=1001(john) gid=1001(john) groups=1001(john),27(sudo),1002(developers)
```

---

# 🛠️ Step 2: File Permissions

## Modify Directory Permissions

```bash
chmod 750 /opt/app
```

### Permission Breakdown

| Permission | Meaning      |
| ---------- | ------------ |
| `7`        | Owner = rwx  |
| `5`        | Group = r-x  |
| `0`        | Others = --- |

Equivalent representation:

```text id="9ckb4m"
rwxr-x---
```

---

## Change Ownership

```bash
chown john:developers /opt/app
```

---

## Verify File Permissions

```bash
ls -la /opt/app
```

---

# 🛠️ Step 3: Service Management (systemd)

## Check Service Status

```bash
systemctl status sshd
```

---

## Enable and Start Service

```bash
systemctl enable --now sshd
```

### Description

| Option   | Purpose                             |
| -------- | ----------------------------------- |
| `enable` | Start service automatically at boot |
| `--now`  | Start service immediately           |

---

## Restart a Service

```bash
systemctl restart nginx
```

---

## View Service Logs

```bash
journalctl -u sshd -n 50 --no-pager
```

### Description

| Option       | Purpose                     |
| ------------ | --------------------------- |
| `-u sshd`    | Filter logs for SSH service |
| `-n 50`      | Display latest 50 entries   |
| `--no-pager` | Print directly to terminal  |

---

# 🛠️ Step 4: Networking

## Display IP Address Information

```bash
ip addr show
```

---

## Display Routing Table

```bash
ip route show
```

---

## View Listening Ports

```bash
ss -tlnp
```

### Description

| Option | Purpose                     |
| ------ | --------------------------- |
| `-t`   | TCP sockets                 |
| `-l`   | Listening sockets           |
| `-n`   | Numeric output              |
| `-p`   | Display process information |

---

## View Network Manager Connections

```bash
nmcli con show
```

# 3.2 Linux Patching — Ubuntu

## 🎯 Objective

Apply operating system updates and automate patch management on Ubuntu systems.

---

# 🛠️ Step 1: Update Package Index & Upgrade

## Update Package Repository Information

```bash
sudo apt update
```

---

## List Available Upgrades

```bash
sudo apt list --upgradable
```

---

## Install Standard Updates

```bash
sudo apt upgrade -y
```

---

## Perform Distribution Upgrade

```bash
sudo apt dist-upgrade -y
```

> ⚠️ `dist-upgrade` may install or remove packages to satisfy dependencies.

---

## Reboot System

```bash
sudo reboot
```

---

# 🛠️ Step 2: Automated Unattended Upgrades

## Install Unattended Upgrades Package

```bash
sudo apt install unattended-upgrades -y
```

---

## Configure Automatic Updates

```bash
sudo dpkg-reconfigure --priority=low unattended-upgrades
```

> 🔄 Enables automatic installation of security updates.

---

# 3.3 Linux Patching — RHEL / CentOS / Rocky

## 🎯 Objective

Apply patches and manage security updates on RHEL-based Linux distributions.

---

# 🛠️ Step 1: Check & Apply Updates

## Check for Available Updates

```bash
sudo dnf check-update
```

---

## Install All Updates

```bash
sudo dnf update -y
```

---

## Install Security Updates Only

```bash
sudo dnf update --security -y
```

> 🔒 Recommended for critical production systems requiring minimal package changes.

---

## Reboot System

```bash
sudo reboot
```

---

# 🛠️ Step 2: List Installed Patches

## Display Package History

```bash
sudo dnf history list
```

---

## List Security Advisories

```bash
sudo dnf updateinfo list security
```

---

# ✅ Best Practice Tips

## 🔒 Security & Patch Management Best Practices

* Always snapshot virtual machines before patching production systems.
* Apply security patches within:

```text id="zqezf8"
72 hours
```

of release in accordance with CIS benchmarks.

* Use automated configuration management tools for consistency.

Recommended tools:

* Ansible
* Red Hat Satellite

---

## 🧪 Testing & Deployment Recommendations

### Follow a Structured Patch Pipeline

```text id="q9h5uh"
Dev → UAT → Prod
```

### Benefits

* Reduces production outages
* Validates application compatibility
* Identifies kernel or dependency conflicts early

---

## 💾 System Maintenance Recommendations

### Maintain Adequate `/boot` Partition Space

Recommended minimum size:

```text id="ww14wr"
1 GB
```

> ⚠️ Required to accommodate kernel updates and rollback options.

* Monitor failed updates and reboot requirements.
* Schedule maintenance windows for production patching.
