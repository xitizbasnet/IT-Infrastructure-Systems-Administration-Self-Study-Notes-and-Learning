# 15. Data Backup — NAS, Veeam, Barracuda, Backblaze & Druva

## 📘 Overview

This document provides enterprise backup configuration procedures for protecting workstation and server data (C: and D: drives) using multiple backup platforms, including NAS-based, cloud-based, and enterprise backup solutions.

Supported systems include:

* Veeam Backup & Replication
* Barracuda Backup
* Backblaze
* Druva inSync

---

# 15.1 Backup Strategy Overview

## 🎯 Objective

Implement a standardized backup strategy across all environments.

---

## 🛠️ Step 1: 3-2-1 Backup Rule

## Backup Principle

| Rule           | Description               |
| -------------- | ------------------------- |
| 3 Copies       | Primary + 2 backups       |
| 2 Media Types  | Different storage systems |
| 1 Offsite Copy | Cloud or remote location  |

> 🛡️ This is the foundational rule for enterprise-grade data protection.

---

# 15.2 NAS Backup (Synology / QNAP Example)

## 🎯 Objective

Configure network-attached storage (NAS) for automated endpoint backups.

---

## 🛠️ Step 1: Install Backup Agent on Windows

### Synology

* Install **Active Backup for Business Agent**

### QNAP

* Install:

  * NetBak Replicator
  * Hyper Data Protector Agent

> 💾 Agents enable endpoint-level backup integration with NAS systems.

---

## 🛠️ Step 2: Create Backup Task

Navigate to NAS UI:

```text id="nas01"
Active Backup → Create Task → PC Backup
```

---

## Backup Configuration

| Setting   | Value                          |
| --------- | ------------------------------ |
| Target    | C: and D: drives               |
| Schedule  | Daily 11:00 PM                 |
| Retention | 30 daily + 12 monthly versions |

> 📦 Versioning enables recovery from historical snapshots.

---

# 15.3 Veeam Backup & Replication

## 🎯 Objective

Configure enterprise-grade backup and recovery using Veeam Backup & Replication.

---

## 🛠️ Step 1: Add Windows Machine

Navigate to:

```text id="veeam01"
Infrastructure → Add → Microsoft Windows
```

---

## Configuration

* Enter hostname or IP
* Provide administrative credentials
* Automatic agent deployment occurs

---

## 🛠️ Step 2: Create Backup Job (Agent-Based)

Navigate to:

```text id="veeam02"
Home → Backup Job → Windows Computer
```

---

## Backup Settings

| Setting     | Value                                       |
| ----------- | ------------------------------------------- |
| Scope       | Entire machine or selected volumes (C:, D:) |
| Destination | NAS or cloud repository                     |
| Schedule    | Daily 10:00 PM                              |
| Full Backup | Weekly (Sunday)                             |

---

## 🧪 Step 3: SureBackup Verification

Navigate to:

```text id="veeam03"
SureBackup Job → Run Verification
```

> ✅ Validates backup integrity and recoverability automatically.

# 15.5 Backblaze Business Backup

---

# 15.6 Druva inSync / Phoenix

## 🎯 Objective

Implement cloud-native endpoint and server backup using Druva inSync.

---

## 🛠️ Step 1: Deploy inSync Agent

* Download agent from Druva Cloud Console
* Install on endpoint
* Auto-registration with cloud platform

---

## 🛠️ Step 2: Configure Backup Sets

Navigate to:

```text id="druva01"
Policies → Backup Policy
```

---

## Configuration

| Setting  | Value                       |
| -------- | --------------------------- |
| Paths    | C:\Users, D:\custom paths   |
| Security | Encryption + Deduplication  |
| Schedule | Continuous or every 4 hours |

---

## 🛠️ Step 3: Druva Phoenix (Server Backup)

* Install Phoenix agent on Windows Server
* Configure volume-level backup
* Enable Air-Gap protection

> 🛡️ Air-gapped backups protect against ransomware deletion attacks.

# 15.7 Verify & Test Restores

## 🎯 Objective

Ensure backup reliability through routine restore testing and reporting.

---

## 🛠️ Step 1: Monthly Restore Test

* Restore random file from each backup system
* Verify integrity via:

  * File open validation
  * Hash comparison

> 🔄 A backup is only valid if restore is successful.

---

## 🛠️ Step 2: Backup Reporting

Monitor:

| Metric        | Threshold       |
| ------------- | --------------- |
| Job Failures  | Alert if >10%   |
| Storage Usage | Alert at >80%   |
| Missed Jobs   | Immediate alert |

# ✅ Best Practice Tips

## 🛡️ Backup Strategy Best Practices

* Follow the **3-2-1 backup rule**
* Maintain at least one offsite copy at all times

---

## 🧪 Testing & Validation

* Perform monthly restore tests
* Treat untested backups as unreliable
* Validate file integrity after restore

---

## 🔐 Security Best Practices

* Enable immutable or air-gapped backups
* Encrypt data at rest and in transit
* Store encryption keys separately from backup systems
* Store credentials in a PAM vault (not plaintext)

---

## 📊 Operational Best Practices

* Define RPO (Recovery Point Objective) and RTO (Recovery Time Objective)
* Exclude temporary and cache directories to reduce backup size
* Monitor backup health dashboards daily
* Investigate failures immediately
