# 12. IAM, PAM, MFA, SSO, MCRA & Okta

## 📘 Overview

This document provides enterprise identity security guidance covering Identity and Access Management (IAM), Privileged Access Management (PAM), Multi-Factor Authentication (MFA), Single Sign-On (SSO), Okta integration, and the Microsoft Cybersecurity Reference Architecture (MCRA).

The guide is aligned with Zero Trust principles and modern identity governance practices.

---

# 12.1 Identity & Access Management (IAM)

## 🎯 Objective

Implement structured identity governance using role-based access control (RBAC) and periodic access reviews.

---

## 🛠️ Step 1: Define RBAC Roles

### Role Design Principle

Map business functions to roles and permissions using:

> 🔐 Least Privilege Access Model

---

## Configure Roles in Microsoft Entra ID

Navigate to:

```text id="iam01"
Entra ID → Roles and Administrators
```

---

## Role Assignment Types

| Type           | Description                      |
| -------------- | -------------------------------- |
| Built-in Roles | Predefined Microsoft roles       |
| Custom Roles   | Organization-defined permissions |

> 📌 Roles should align with job functions, not individuals.

---

## 🛠️ Step 2: Access Reviews

## Create Access Review

Navigate to:

```text id="iam02"
Entra ID → Identity Governance → Access Reviews → New
```

---

## Review Strategy

| Setting   | Value            |
| --------- | ---------------- |
| Frequency | Quarterly        |
| Scope     | Privileged roles |

> 🔍 Regular access reviews reduce privilege creep and ensure compliance.

---

# 12.2 Privileged Access Management (PAM)

## 🎯 Objective

Secure administrative access using Just-In-Time (JIT) privilege elevation.

---

# 🛠️ Step 1: Enable Entra PIM

## Configure Privileged Identity Management

Navigate to:

```text id="pam01"
Entra ID → Privileged Identity Management
```

---

## Role Configuration

| Setting      | Recommendation                  |
| ------------ | ------------------------------- |
| Global Admin | Set as Eligible (not Permanent) |
| Activation   | Requires MFA + justification    |

> 🛡️ Reduces risk of persistent administrative privileges.

---

# 🛠️ Step 2: Just-In-Time (JIT) Access

## Activate Privileged Role

Navigate to:

```text id="pam02"
PIM → My Roles → Activate
```

---

## Activation Parameters

| Setting       | Value                       |
| ------------- | --------------------------- |
| Duration      | Max 8 hours                 |
| Approval      | Required (where configured) |
| Justification | Mandatory                   |

> ⏱️ Temporary elevation reduces attack surface exposure.

---

# 12.3 MFA Configuration

## 🎯 Objective

Enforce strong authentication mechanisms across all user accounts.

---

# 🛠️ Step 1: Enforce MFA via Conditional Access

## Create Policy

Navigate to:

```text id="mfa01"
Entra ID → Conditional Access → New Policy
```

---

## Policy Configuration

| Setting       | Value       |
| ------------- | ----------- |
| Users         | All Users   |
| Grant Control | Require MFA |

---

## Exclusions

* Break-glass accounts (emergency access accounts)

> 🚨 Ensure break-glass accounts are monitored and secured separately.

---

# 🛠️ Step 2: Authentication Methods

## Configure Authentication Methods Policy

Navigate to:

```text id="mfa02"
Entra ID → Security → Authentication Methods
```

---

## Enabled Methods

| Method                         | Status                             |
| ------------------------------ | ---------------------------------- |
| Microsoft Authenticator (Push) | Enabled                            |
| FIDO2 Security Keys            | Enabled                            |
| SMS Authentication             | Disabled (for privileged accounts) |

> 🔐 Hardware-based authentication is preferred for high-privilege accounts.

# ✅ Best Practice Tips

## 🔐 Zero Trust Security Model

> 🧭 Core Principle:

* Verify explicitly
* Use least privilege
* Assume breach

---

## 🚨 Privileged Account Security

* Break-glass accounts must:

  * Bypass Conditional Access
  * Be heavily monitored 24/7
* Audit privileged role assignments monthly via PIM logs

---

## 📊 Identity Security Monitoring

* Enable Entra ID Protection risk-based policies
* Block or challenge risky sign-ins automatically
* Monitor authentication anomalies continuously

---

## 🔑 Password Security Controls

* Enforce Entra ID Password Protection
* Block weak and commonly used passwords
* Extend policy to on-prem Active Directory environments

---

## ⚙️ Governance Recommendations

* Conduct regular access reviews
* Remove inactive or unnecessary privileges
* Apply consistent role-based access models across all systems
