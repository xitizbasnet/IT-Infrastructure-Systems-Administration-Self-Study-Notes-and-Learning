# 05. Hyper-V Virtualisation

## 📘 Overview

This document provides step-by-step procedures for installing, configuring, and managing Hyper-V virtual machines and networking within a Windows Server environment.

The guide covers:

* Enabling the Hyper-V role
* Creating virtual switches
* Deploying virtual machines
* Configuring VM settings
* Managing checkpoints and live migration
* Hyper-V operational best practices

---

# 5.1 Enable Hyper-V Role

## 🎯 Objective

Install and configure the Hyper-V virtualization platform on Windows Server.

---

# 🛠️ Step 1: Enable Hyper-V via Server Manager

## Open Add Roles & Features Wizard

Navigate to:

```text id="rm8m4x"
Server Manager → Add Roles & Features
```

---

## Install Hyper-V Role

Select:

```text id="y70p7e"
Hyper-V
```

### Installation Actions

* Accept all sub-features
* Include management tools
* Proceed with installation

---

## Complete Installation

Select:

```text id="mjlwmc"
Install
```

Reboot the server when prompted.

> 🔄 A restart is required to complete Hyper-V installation.

---

# 🛠️ Step 2: Enable Hyper-V via PowerShell

## Install Hyper-V Role

```powershell id="0m3q1x"
Install-WindowsFeature -Name Hyper-V -IncludeManagementTools -Restart
```

---

## Parameter Breakdown

| Parameter                 | Description                                      |
| ------------------------- | ------------------------------------------------ |
| `-Name Hyper-V`           | Installs Hyper-V role                            |
| `-IncludeManagementTools` | Installs Hyper-V management utilities            |
| `-Restart`                | Automatically restarts server after installation |

---

# 5.2 Create a Virtual Switch

## 🎯 Objective

Configure virtual networking to allow virtual machines network connectivity.

---

# 🛠️ Step 1: Create External Virtual Switch

## Open Virtual Switch Manager

Navigate to:

```text id="92m0y0"
Hyper-V Manager → Virtual Switch Manager
```

---

## Create New Virtual Switch

Select:

```text id="1g8br7"
New → External
```

Choose the physical network adapter (NIC), then click:

```text id="1ujn1m"
Apply
```

---

## Configure Virtual Switch

### Example Configuration

| Setting | Value                         |
| ------- | ----------------------------- |
| Name    | `ExternalSwitch`              |
| Type    | External                      |
| Purpose | VM access to LAN and Internet |

> 🌐 External virtual switches bridge VMs to the physical network.

---

# 5.3 Create a Virtual Machine

## 🎯 Objective

Deploy and configure a new Hyper-V virtual machine.

---

# 🛠️ Step 1: New VM Wizard

## Launch VM Creation Wizard

Navigate to:

```text id="v3i3mx"
Hyper-V Manager → New → Virtual Machine
```

---

## Configure Virtual Machine Settings

### Recommended Configuration

| Setting            | Recommendation              |
| ------------------ | --------------------------- |
| Generation         | Generation 2 (UEFI)         |
| Memory             | Minimum 2 GB                |
| Memory Type        | Dynamic Memory Recommended  |
| Network            | Connect to `ExternalSwitch` |
| Virtual Disk       | New VHD (Example: 60 GB)    |
| Installation Media | Attach ISO image            |

---

## Example Virtual Disk

```text id="7glrl8"
60 GB
```

---

# 🛠️ Step 2: VM Settings — Best Configuration

## Linux Guest Configuration

For Linux virtual machines:

* Disable Secure Boot

OR

* Change Secure Boot template to:

```text id="a6thwe"
Microsoft UEFI Certificate Authority
```

> 🐧 Required for compatibility with many Linux distributions.

---

## Enable Integration Services

Navigate to:

```text id="qlxk1w"
Integration Services → Guest Services
```

Enable Guest Services integration.

---

## Configure Processor Settings

### Recommended Actions

* Increase processor count as required
* Enable:

```text id="bq9y3l"
NUMA Spanning
```

for large virtual machines.

> ⚡ NUMA spanning improves resource allocation flexibility for larger workloads.

# 5.4 Checkpoints & Live Migration

## 🎯 Objective

Manage VM recovery points and perform workload migrations between Hyper-V hosts.

---

# 🛠️ Step 1: Create Checkpoint

## Create VM Snapshot

```powershell id="5xv5uv"
Checkpoint-VM -Name 'Server01' -SnapshotName 'Pre-Patch-$(Get-Date -f yyyyMMdd)'
```

---

## Purpose of Checkpoints

Checkpoints provide:

* Rollback capability
* Pre-maintenance recovery
* Testing and troubleshooting snapshots

> ⚠️ Avoid long-term retention of production checkpoints to prevent storage growth.

---

# 🛠️ Step 2: Configure Live Migration

## Live Migration Requirements

Both Hyper-V hosts must:

* Belong to the same Active Directory domain
* Have access to shared storage
* Support compatible processor configurations

---

## Enable Live Migration

Navigate to:

```text id="dbfqeh"
Hyper-V Settings → Live Migrations
```

Enable:

```text id="pdjicj"
Enable Incoming and Outgoing Live Migrations
```

Configure the concurrent migration limit as appropriate for the environment.

> 🔄 Live Migration enables VM movement with minimal downtime.

---

# ✅ Best Practice Tips

## 🖥️ Virtual Machine Best Practices

* Use Generation 2 VMs for all new Windows and Linux workloads.
* Allocate virtual CPUs based on workload requirements.
* Avoid overcommitting memory and CPU resources.

---

## 💾 Storage Recommendations

* Store VHD/VHDX files on separate storage volumes from the host operating system.
* Use high-performance storage for production workloads.
* Monitor virtual disk growth regularly.

---

## ⚙️ Memory Management Recommendations

### Dynamic Memory Usage

Recommended for:

* Desktop VMs
* Test environments
* Development systems

Avoid for:

* Production database servers
* High-performance workloads
* Latency-sensitive applications

---

## 📸 Checkpoint Management Recommendations

* Schedule automatic checkpoint cleanup.
* Remove outdated snapshots regularly.
* Use production checkpoints where supported.

> ⚠️ Excessive checkpoints can cause disk bloat and performance degradation.

---

## 🔐 Hyper-V Security & Administration Recommendations

* Join Hyper-V hosts to Active Directory domains.
* Configure delegated administration for Hyper-V management.
* Restrict Hyper-V Manager access using role-based permissions.
* Enable regular backup and disaster recovery testing.
