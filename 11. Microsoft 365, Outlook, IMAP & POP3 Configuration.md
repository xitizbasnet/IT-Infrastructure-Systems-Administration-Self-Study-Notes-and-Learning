# 11. Microsoft 365, Outlook, IMAP & POP3 Configuration

## 📘 Overview

This document provides configuration guidance for Microsoft 365 tenant setup, Exchange Online administration, and email client configuration using IMAP, POP3, and SMTP protocols.

The guide covers:

* Microsoft 365 tenant setup and domain verification
* Licensing assignment
* Exchange Online mailbox management
* Outlook IMAP/POP3 configuration
* SMTP relay configuration for devices and applications
* Email security best practices

---

# 11.1 Microsoft 365 Tenant Initial Setup

## 🎯 Objective

Configure a new Microsoft 365 tenant with custom domain integration and user licensing.

---

# 🛠️ Step 1: Add & Verify Custom Domain

## Add Domain in Microsoft 365

Navigate to:

```text id="m365dom01"
Microsoft 365 Admin Centre → Settings → Domains → Add domain
```

---

## DNS Verification Requirements

Add the following DNS records at your domain registrar:

| Record Type | Purpose                      |
| ----------- | ---------------------------- |
| TXT         | Domain verification          |
| MX          | Mail routing                 |
| CNAME       | Autodiscover service         |
| SPF         | Sender validation            |
| DKIM        | Email signing authentication |

> 🔐 Proper DNS configuration ensures secure and reliable email delivery.

---

# 🛠️ Step 2: Assign Licences

## Assign Microsoft 365 Licenses

Navigate to:

```text id="m365lic01"
Admin Centre → Billing → Licences
```

---

## Assignment Options

* Assign to individual users
* Assign via group-based licensing

> 📦 Licenses enable access to Exchange Online, Teams, SharePoint, and other services.

# 11.2 Exchange Online — Shared Mailbox & Distribution List

## 🎯 Objective

Configure shared mailboxes and distribution groups in Exchange Online.

---

# 🛠️ Step 1: Create Shared Mailbox

## Create Mailbox

Navigate to:

```text id="exo01"
Exchange Admin Centre → Recipients → Mailboxes → Add shared mailbox
```

---

## Permissions Configuration

Assign:

* Full Access
* Send As permissions

> 👥 Shared mailboxes are used for team-based communication (e.g., [support@company.com](mailto:support@company.com)).

---

# 🛠️ Step 2: Create Distribution Group

## Create Group

Navigate to:

```text id="exo02"
Exchange Admin Centre → Recipients → Groups → Add distribution group
```

---

## Configuration Options

| Setting               | Description                        |
| --------------------- | ---------------------------------- |
| Alias                 | Email identifier                   |
| Owner                 | Group administrator                |
| Members               | Recipients list                    |
| Delivery Restrictions | Control inbound/outbound messaging |

> 📬 Distribution groups simplify mass email communication.

---

# 11.3 Outlook — Manual IMAP / POP3 Configuration

## 🎯 Objective

Manually configure email accounts using IMAP and POP3 protocols in Outlook.

---

# 🛠️ Step 1: Configure IMAP in Outlook

## Account Setup

Navigate to:

```text id="outlookimap01"
File → Add Account → Advanced Setup → IMAP
```

---

## IMAP Settings

| Setting         | Value               |
| --------------- | ------------------- |
| Incoming Server | imap.yourdomain.com |
| Port            | 993                 |
| Security        | SSL                 |
| Outgoing Server | smtp.yourdomain.com |
| Port            | 587                 |
| Security        | TLS                 |

---

## Authentication

| Setting  | Value              |
| -------- | ------------------ |
| Username | Full email address |
| Password | Mailbox password   |

---

> 📥 IMAP synchronizes mail across multiple devices.

---

# 🛠️ Step 2: Configure POP3 in Outlook

## POP Account Setup

Navigate to:

```text id="outlookpop01"
File → Add Account → Advanced Setup → POP
```

---

## POP3 Settings

| Setting         | Value               |
| --------------- | ------------------- |
| Incoming Server | pop.yourdomain.com  |
| Port            | 995                 |
| Security        | SSL                 |
| Outgoing Server | smtp.yourdomain.com |
| Port            | 587                 |
| Security        | TLS                 |

---

## Recommended Configuration

> 📌 Enable:

* Leave messages on server for 30 days

This prevents data loss when accessing mail from multiple devices.

# 11.4 Microsoft 365 SMTP Relay (Devices & Applications)

## 🎯 Objective

Configure SMTP relay for printers, scanners, and applications.

---

# 🛠️ Step 1: Enable SMTP AUTH for Mailbox

## PowerShell Configuration

```powershell id="smtp01"
Set-CASMailbox -Identity scanner@corp.com -SmtpClientAuthenticationDisabled $false
```

---

## Purpose

Enables SMTP authentication for sending mail via applications or devices.

---

# 🛠️ Step 2: Direct Send (No Authentication)

## SMTP Configuration

| Setting     | Value                                      |
| ----------- | ------------------------------------------ |
| SMTP Server | yourtenant-com.mail.protection.outlook.com |
| Port        | 25                                         |
| Encryption  | None                                       |

---

## Requirements

* Must originate from trusted IPs
* Must be within accepted domains

> 📤 Used for internal devices like scanners and printers.

# ✅ Best Practice Tips

## 🔐 Email Security Best Practices

* Enable DKIM and DMARC for all sending domains
* Enforce Multi-Factor Authentication (MFA) for all users
* Consider passwordless authentication methods

---

## 🛡️ Legacy Protocol Management

* Block legacy authentication protocols (IMAP/POP3) via Conditional Access for standard users
* Allow exceptions only for approved service accounts

---

## 📡 Mail Flow & Monitoring

* Use Message Trace in Exchange Admin Centre to monitor email delivery
* Configure alerts for mail flow failures or anomalies

---

## ⚙️ Configuration Best Practices

* Configure Autodiscover SRV records for seamless Outlook setup
* Prefer IMAP over POP3 for multi-device environments
* Standardize SMTP relay usage for network devices
