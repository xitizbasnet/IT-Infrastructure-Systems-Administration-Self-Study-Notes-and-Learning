# 14. Scanner Configuration via SMTP & Fax Settings

## 📘 Overview

This document provides standardized configuration procedures for enabling enterprise multifunction devices (MFDs) and scanners to support:

* Scan-to-Email (SMTP)
* Scan-to-Network Folder (SMB)
* Fax configuration (traditional and VoIP-based)

These configurations are commonly used in enterprise environments to digitize document workflows and improve operational efficiency.

---

# 14.1 Scan-to-Email via SMTP (Port 587)

## 🎯 Objective

Configure a scanner or multifunction device to send scanned documents via email using SMTP authentication.

---

## 🛠️ Step 1: Configure SMTP on Printer / MFD

Access the device web interface:

```text id="scanemail01"
http://<printer-ip>
```

---

Navigate to:

```text id="scanemail02"
Network → Email → SMTP Settings
```

> ⚠️ Menu names may vary depending on manufacturer.

---

# 🛠️ Step 2: SMTP Settings — Microsoft 365 / Exchange Online

Configure the following:

| Setting        | Value                                       |
| -------------- | ------------------------------------------- |
| SMTP Server    | smtp.office365.com                          |
| Port           | 587                                         |
| Encryption     | STARTTLS                                    |
| Authentication | Enabled                                     |
| Username       | [scanner@corp.com](mailto:scanner@corp.com) |
| Password       | App Password                                |
| From Address   | [scanner@corp.com](mailto:scanner@corp.com) |

> 📧 The “From” address must match the authenticated mailbox.

---

## 🔐 Requirements

* App Password required for MFA-enabled accounts
* SMTP AUTH must be enabled for the mailbox

---

# 🛠️ Step 3: SMTP Settings — Gmail Relay

| Setting        | Value                 |
| -------------- | --------------------- |
| SMTP Server    | smtp.gmail.com        |
| Port           | 587                   |
| Encryption     | TLS                   |
| Authentication | App Password required |

> 🔑 Google App Passwords are generated from account security settings.

---

# 🛠️ Step 4: SMTP Settings — Internal Exchange Relay (No Authentication)

| Setting        | Value                           |
| -------------- | ------------------------------- |
| SMTP Server    | your-exchange-server.corp.local |
| Port           | 25                              |
| Authentication | None                            |

---

## Exchange Configuration Requirement

* Add printer IP to **Receive Connector**
* Allow anonymous relay from trusted devices only

> 🛡️ Restrict relay access to prevent abuse of SMTP services.

---

# 🧪 Step 5: Test Scan-to-Email

* Perform test scan from device panel
* Verify email delivery in mailbox

> 📤 Successful delivery confirms SMTP configuration is functional.

---

# 14.2 Scan-to-Folder (SMB)

## 🎯 Objective

Configure scanners to store documents directly into a network shared folder.

---

# 🛠️ Step 1: Create Shared Folder on Server

## PowerShell Command

```powershell id="scanfolder01"
New-SmbShare -Name "Scans" -Path "D:\Scans" -FullAccess "CORP\ScannerSvc"
```

---

## Configuration Details

| Setting    | Value                   |
| ---------- | ----------------------- |
| Share Name | Scans                   |
| Path       | D:\Scans                |
| Access     | Scanner service account |

> 📁 Dedicated service accounts improve security and traceability.

---

# 🛠️ Step 2: Configure Printer Scan Settings

Navigate to:

```text id="scanfolder02"
Scan → Scan to Network Folder
```

---

## SMB Configuration

| Setting        | Value                      |
| -------------- | -------------------------- |
| Path           | \server01\Scans            |
| Protocol       | SMB                        |
| Authentication | ScannerSvc service account |

> 📂 SMB scanning enables direct file storage on network servers.

---

# 14.3 Enable Fax (If Hardware Supported)

## 🎯 Objective

Configure fax functionality for multifunction devices supporting PSTN or VoIP faxing.

---

# 🛠️ Step 1: Check Fax Hardware Support

Ensure device supports:

* RJ-11 phone line connection
* Built-in fax modem

> 📞 Hardware capability is required for traditional fax functionality.

---

# 🛠️ Step 2: Configure Fax — Windows Fax & Scan

Navigate to:

```text id="fax01"
Start → Windows Fax and Scan → Tools → Fax Accounts
```

---

## Setup Options

* Add fax modem
* Configure sender details
* Set line configuration

---

# 🛠️ Step 3: Printer Fax Configuration

Navigate to:

```text id="fax02"
Printer Web UI → Fax Settings
```

---

## Fax Configuration Parameters

| Setting             | Value                      |
| ------------------- | -------------------------- |
| Line Type           | PSTN or VoIP (FoIP)        |
| Fax Number          | Assigned DDI number        |
| ECM                 | Enabled                    |
| Junk Fax Protection | Enabled                    |
| Resolution          | Fine (200 DPI recommended) |

> 📠 ECM reduces transmission errors and improves reliability.

---

# 🛠️ Step 4: Fax-to-Email (T.38 / VoIP)

## Configuration

* Enable SMTP relay on MFD (same as scan-to-email setup)
* Route incoming faxes to mailbox as PDF attachments

> 📧 Fax-to-email improves archiving and accessibility.

# ✅ Best Practice Tips

## 🔐 Security Best Practices

* Use dedicated service accounts for scanner authentication
* Never use administrative credentials for device integration
* Prefer port **587 with STARTTLS** over port 25

---

## 📧 Email Security Recommendations

* Enable SPF, DKIM, and DMARC to prevent spoofing
* Monitor scan-to-email logs for anomalies
* Ensure scanned emails do not bypass security filtering

---

## 📁 File Storage Best Practices

* Store scans in restricted-access network shares
* Avoid public or open file shares
* Apply NTFS permissions for strict access control

---

## 📠 Fax & Compliance Best Practices

* Prefer fax-to-email over physical fax lines for auditability
* Ensure ECM is enabled for reliable transmission
* Test fax routing with known endpoints before production use

---

## ⚙️ Operational Best Practices

* Use centralized logging for scan and fax activity
* Regularly audit service accounts used by MFDs
* Validate SMTP and SMB configurations after firmware updates
